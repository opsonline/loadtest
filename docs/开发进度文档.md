## 📊 项目概况

基于 Web 的压测管理系统，支持 HAR 文件导入、场景管理、压测执行、报告分析等功能。

### 技术栈
- **后端**: Django 4.2 + DRF + Locust + Celery
- **前端**: Vue 3 + Element Plus + Vite
- **数据库**: SQLite (开发) / MySQL (生产)

## 🚀 快速启动

### 后端启动

```bash
# 1. 进入后端目录
cd backend

# 2. 激活虚拟环境
source venv/bin/activate

# 3. 安装依赖
pip install -r requirements.txt

# 4. 执行数据库迁移
python manage.py migrate

# 5. 创建超级用户 (可选)
python manage.py createsuperuser

# 6. 启动开发服务器
python manage.py runserver
```

服务运行在: http://localhost:8000

### 前端启动

```bash
# 1. 进入前端目录
cd frontend

# 2. 安装依赖
npm install

# 3. 启动开发服务器
npm run dev
```

服务运行在: http://localhost:3000

### Celery 启动 (可选，用于异步任务)

```bash
# Worker
celery -A config worker -l info

# Beat (定时任务)
celery -A config beat -l info
```

## 📝 数据库模型

### users.User
- 自定义用户模型，扩展 AbstractUser
- 字段: id, username, email, phone, avatar, is_active

### scenarios.Scenario
- 压测场景
- 字段: id, name, description, created_by, is_imported_from_har, default_users, default_spawn_rate, default_duration

### scenarios.Request
- 场景请求
- 字段: id, scenario, name, method, url, headers, body_type, body, weight, think_time, timeout, datasource, order

### scenarios.HARImport
- HAR 导入记录
- 字段: id, scenario, file_name, file_path, resource_types, host_replacement, imported_count

### reports.Report
- 压测报告
- 字段: id, scenario, name, status, users, spawn_rate, duration, total_requests, success_rate, avg_response_time, rps, 等

### reports.RequestStats
- 请求级别统计
- 字段: id, report, request_name, method, num_requests, avg_response_time, min/max/p50/p90/p95/p99

### environments.Environment
- 环境配置
- 字段: id, name, description, created_by, is_default

### environments.Variable
- 环境变量
- 字段: id, environment, name, value, var_type, scope

### datasources.DataSource
- 数据源
- 字段: id, name, source_type, file_path, db_host, db_port, db_user, db_password, db_name, python_script, 等

### api_tests.TestSuite
- 测试套件
- 字段: id, name, description, created_by

### api_tests.TestCase
- 测试用例
- 字段: id, suite, name, method, url, headers, body, order

### api_tests.Assertion
- 断言规则
- 字段: id, test_case, name, assertion_type, target_path, expected_value, operator, min_value, max_value

### api_tests.TestResult
- 测试结果
- 字段: id, test_case, status, response_status, response_time, assertion_results, error_message

## 🔧 配置说明

### 后端配置 (backend/config/settings.py)

```python
# 数据库配置
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.sqlite3',
        'NAME': BASE_DIR / 'db.sqlite3',
    }
}

# JWT 配置
JWT_SECRET_KEY = os.getenv('JWT_SECRET_KEY', SECRET_KEY)
JWT_ACCESS_TOKEN_LIFETIME = 60 * 60 * 24  # 24 小时
JWT_REFRESH_TOKEN_LIFETIME = 60 * 60 * 24 * 7  # 7 天

# Celery 配置
CELERY_BROKER_URL = os.getenv('CELERY_BROKER_URL', 'redis://localhost:6379/0')
CELERY_RESULT_BACKEND = os.getenv('CELERY_RESULT_BACKEND', 'redis://localhost:6379/0')
```

### 前端配置 (frontend/vite.config.js)

```javascript
server: {
  port: 3000,
  proxy: {
    '/api': {
      target: 'http://localhost:8000',
      changeOrigin: true,
    },
  },
}
```
