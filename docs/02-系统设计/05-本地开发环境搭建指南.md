# 本地开发环境搭建指南

## 一、环境要求

### 1.1 基础软件

| 软件 | 版本要求 | 用途 |
|------|----------|------|
| Python | 3.11+ | 后端开发语言 |
| Node.js | 18+ | 前端开发工具 |
| Git | 2.0+ | 版本控制 |
| VS Code | 最新版 | 推荐编辑器 |
| PyCharm | 最新版 | 推荐 Python IDE |
| MySQL | 8.0+ | 数据库（可选，SQLite 可用于开发） |
| Redis | 7.0+ | 缓存和消息队列（可选） |

### 1.2 操作系统

| 系统 | 支持情况 |
|------|----------|
| Windows 10/11 | WSL2 推荐 |
| macOS | 完全支持 |
| Ubuntu 20.04+ | 完全支持 |

## 二、Windows WSL2 环境搭建

### 2.1 安装 WSL2

```powershell
# PowerShell (管理员模式)
# 1. 启用 WSL 功能
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart

# 2. 启用虚拟机平台
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart

# 3. 下载并安装 Linux 内核更新包
# https://wslstorestorage.blob.core.windows.net/wslblob/wsl_update_x64.msi

# 4. 设置 WSL2 为默认版本
wsl --set-default-version 2

# 5. 安装 Ubuntu 20.04
wsl --install -d Ubuntu-20.04
```

### 2.2 配置 WSL2 开发环境

```bash
# 打开 Ubuntu 终端

# 1. 更新系统
sudo apt update && sudo apt upgrade -y

# 2. 安装 Python 开发环境
sudo apt install -y python3 python3-pip python3-venv python3-dev build-essential

# 3. 安装 Node.js
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs

# 4. 安装 Git
sudo apt install -y git

# 5. 安装 Docker (可选)
curl -fsSL https://get.docker.com -s | sh
sudo usermod -aG docker $USER

# 6. 安装 MySQL 客户端
sudo apt install -y mysql-client

# 7. 安装 Redis (可选)
sudo apt install -y redis-server

# 8. 安装 Python 虚拟环境管理
pip install virtualenv
```

### 2.3 配置 Windows 终端

```json
// settings.json
{
    "profiles": {
        "list": [
            {
                "name": "Ubuntu-20.04",
                "commandline": "wsl.exe -d Ubuntu-20.04",
                "startingDirectory": "//wsl$/Ubuntu-20.04/home/$USER",
                "icon": "ms-appx:///Images/Linux.png"
            }
        ]
    }
}
```

## 三、macOS 环境搭建

### 3.1 使用 Homebrew 安装

```bash
# 1. 安装 Homebrew
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 2. 安装 Python
brew install python@3.11

# 3. 安装 Node.js
brew install node@18

# 4. 安装 Git
brew install git

# 5. 安装 MySQL (可选)
brew install mysql@8.0

# 6. 安装 Redis (可选)
brew install redis

# 7. 配置环境变量
echo 'export PATH="/usr/local/opt/python@3.11/bin:$PATH"' >> ~/.zshrc
echo 'export PATH="/usr/local/opt/node@18/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

## 四、项目初始化

### 4.1 克隆项目

```bash
# 1. 克隆代码仓库
git clone <repository-url> loadtest
cd loadtest

# 2. 创建项目分支
git checkout -b develop

# 3. 配置 Git 用户信息
git config user.name "Your Name"
git config user.email "your.email@example.com"
```

### 4.2 后端环境搭建

```bash
# 1. 创建虚拟环境
python -m venv .venv

# 2. 激活虚拟环境
# Linux/macOS
source .venv/bin/activate
# Windows (WSL)
source .venv/Scripts/activate
# Windows (CMD)
.venv\Scripts\activate

# 3. 升级 pip
pip install --upgrade pip

# 4. 安装 Python 依赖
pip install -r requirements.txt

# 5. 复制环境变量文件
cp .env.example .env

# 6. 编辑环境变量
nano .env
```

### 4.3 前端环境搭建

```bash
# 1. 进入前端目录
cd frontend

# 2. 安装 Node.js 依赖
npm install

# 3. 配置环境变量
cp .env.example .env.local
```

### 4.4 环境变量配置示例

```bash
# .env 文件配置

# ============ 数据库配置 (SQLite 用于开发) ============
DEBUG=1
SECRET_KEY=your-dev-secret-key-change-in-production

# SQLite 配置
DATABASE_ENGINE=sqlite
DATABASE_NAME=db.sqlite3

# MySQL 配置 (可选)
# DATABASE_ENGINE=mysql
# DATABASE_NAME=loadtest
# DATABASE_USER=loadtest
# DATABASE_PASSWORD=your_password
# DATABASE_HOST=localhost
# DATABASE_PORT=3306

# ============ Redis 配置 (可选，开发时可禁用) ============
REDIS_URL=redis://localhost:6379/0

# ============ Celery 配置 ============
CELERY_BROKER_URL=redis://localhost:6379/0
CELERY_RESULT_BACKEND=redis://localhost:6379/1

# ============ JWT 配置 ============
JWT_SECRET_KEY=your-dev-jwt-secret-key
JWT_EXPIRATION_HOURS=24

# ============ CORS 配置 ============
CORS_ALLOWED_ORIGINS=http://localhost:5173,http://localhost:8000
```

```bash
# .env.local 文件配置

# API 基础地址
VITE_API_BASE_URL=http://localhost:8000/api/v1

# WebSocket 地址
VITE_WS_URL=ws://localhost:8000/ws
```

## 五、数据库配置

### 5.1 使用 SQLite (默认)

```python
# settings.py
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.sqlite3',
        'NAME': BASE_DIR / 'db.sqlite3',
    }
}
```

### 5.2 使用 MySQL

```python
# settings.py
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.mysql',
        'NAME': 'loadtest',
        'USER': 'loadtest',
        'PASSWORD': 'your_password',
        'HOST': 'localhost',
        'PORT': '3306',
        'OPTIONS': {
            'charset': 'utf8mb4',
        },
    }
}
```

### 5.3 创建数据库

```bash
# MySQL 方式
mysql -u root -p
CREATE DATABASE loadtest CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
GRANT ALL PRIVILEGES ON loadtest.* TO 'loadtest'@'localhost';
FLUSH PRIVILEGES;
```

## 八、代码规范工具

### 8.1 Python 代码检查

```bash
# 安装检查工具
pip install flake8 black isort mypy

# 使用 black 格式化代码
black .

# 使用 isort 排序 imports
isort .

# 使用 flake8 检查代码
flake8 .

# 使用 mypy 检查类型
mypy .
```

### 8.2 前端代码检查

```bash
# 安装检查工具
npm install -D eslint @vue/eslint-config-prettier

# 检查代码
npm run lint

# 修复格式问题
npm run lint -- --fix
```

### 8.3 pre-commit 钩子

```bash
# 安装 pre-commit
pip install pre-commit

# 创建配置文件
cat > .pre-commit-config.yaml << EOF
repos:
  - repo: https://github.com/psf/black
    rev: 23.12.1
    hooks:
      - id: black
        language_version: python3.11

  - repo: https://github.com/pycqa/isort
    rev: 5.13.2
    hooks:
      - id: isort

  - repo: https://github.com/pycqa/flake8
    rev: 7.0.0
    hooks:
      - id: flake8

  - repo: https://github.com/pre-commit/pre-commit-hooks
    rev: v4.5.0
    hooks:
      - id: trailing-whitespace
      - id: end-of-file-fixer
      - id: check-yaml
      - id: check-added-large-files
EOF

# 安装钩子
pre-commit install
```

## 十、常见问题

### 10.1 Python 依赖安装失败

```bash
# 清理 pip 缓存
pip cache purge

# 升级 pip
pip install --upgrade pip

# 使用镜像源安装
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple/
```

### 10.2 Node.js 版本问题

```bash
# 使用 nvm 管理 Node 版本
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.bashrc
nvm install 18
nvm use 18
```

### 10.3 数据库连接问题

```bash
# 检查 MySQL 服务状态
sudo systemctl status mysql

# 检查端口占用
netstat -tlnp | grep 3306

# 检查防火墙
sudo ufw status
```

### 10.4 端口被占用

```bash
# 查找占用端口的进程
lsof -i :8000
# 或
netstat -tlnp | grep 8000

# 终止进程
kill -9 <PID>
```

## 十一、开发工作流

### 11.1 日常开发流程

```bash
# 1. 每天开始工作时拉取最新代码
git checkout develop
git pull origin develop

# 2. 创建功能分支
git checkout -b feature/xxx

# 3. 开发功能
# ... 编辑代码 ...

# 4. 运行测试
python manage.py test

# 5. 提交代码
git add .
git commit -m "feat: xxx"

# 6. 推送代码
git push origin feature/xxx

# 7. 创建 Pull Request
```

### 11.2 代码审查

1. 提交 Pull Request
2. 至少一名团队成员审查代码
3. 通过 CI/CD 检查
4. 合并到 develop 分支
5. 删除功能分支
