# 压测系统项目需求说明文档

## 一、项目概述

### 1.1 项目背景

随着微服务架构的普及，系统性能测试成为保障服务质量的重要环节。本项目旨在开发一套基于 Web 的压测管理平台，支持将 HAR（HTTP Archive）文件转换为 Locust 压测场景，提供完整的压测管理、报告分析、环境变量管理等功能，帮助开发团队高效完成性能测试工作。

### 1.2 项目目标

构建一套完整的压测管理平台，实现以下核心目标：

- 提供直观的 Web 管理界面，降低压测使用门槛
- 支持 HAR 文件一键导入，快速生成压测场景
- 提供完整的压测报告管理，支持历史报告对比分析
- 支持环境变量管理，实现多环境灵活切换
- 支持多种数据源，满足复杂业务场景
- 集成接口测试功能，兼顾功能验证与性能测试

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| HAR | HTTP Archive，一种记录 HTTP 交互的 JSON 格式文件 |
| Locust | Python 编写的分布式压测框架 |
| 场景 | 压测场景的抽象，包含一组 HTTP 请求及配置 |
| RPS | Requests Per Second，每秒请求数 |
| P99 | 第 99 百分位响应时间 |

## 二、功能需求

### 2.1 接口测试功能

#### 2.1.1 功能描述

提供独立的接口测试功能，支持单个接口或接口套件的验证测试，类似于 Postman 的基本功能。

#### 2.1.2 功能列表

| 功能模块 | 功能描述 | 优先级 |
|----------|----------|--------|
| 测试套件管理 | 创建、编辑、删除测试套件 | 高 |
| 请求配置 | 配置 HTTP 请求方法、URL、Headers、Body | 高 |
| 断言规则 | 支持状态码、响应时间、JSON 路径、正则等断言 | 高 |
| 测试执行 | 执行单个请求或整个测试套件 | 高 |
| 结果展示 | 展示测试通过/失败状态、响应详情 | 高 |
| 结果历史 | 保存历史测试结果，支持查看 | 中 |

#### 2.1.3 断言规则类型

系统应支持以下断言类型：

| 断言类型 | 说明 | 示例 |
|----------|------|------|
| 状态码 | 验证 HTTP 响应状态码 | status_code eq 200 |
| 响应时间 | 验证响应时间阈值 | response_time lt 1000 |
| JSON 路径 | 验证 JSON 响应特定字段 | json_path(data.user.id) eq 1001 |
| 正则匹配 | 使用正则表达式验证响应 | regex(token, ^eyJ.*$) |
| 包含验证 | 验证响应包含特定内容 | contains(response.body, "success") |
| 数值范围 | 验证数值在特定范围内 | numeric_field gte 0 and lte 100 |

### 2.2 压测报告管理

#### 2.2.1 功能描述

管理和展示压测执行结果，提供多维度数据分析。

#### 2.2.2 功能列表

| 功能模块 | 功能描述 | 优先级 |
|----------|----------|--------|
| 报告列表 | 展示所有压测报告，支持筛选排序 | 高 |
| 报告详情 | 展示单次压测的详细统计数据 | 高 |
| 统计概览 | 展示总请求数、RPS、错误率、响应时间等 | 高 |
| 请求级别统计 | 展示每个请求的独立统计数据 | 高 |
| 图表分析 | 提供 RPS、响应时间、错误率等图表展示 | 高 |
| 报告对比 | 支持对比两次压测的结果 | 中 |
| 报告导出 | 导出报告为 JSON、PDF 格式 | 中 |
| 报告删除 | 删除不需要的历史报告 | 低 |

#### 2.2.3 统计指标

压测报告应包含以下统计指标：

| 指标 | 说明 |
|------|------|
| 总请求数 | 压测期间发起的总请求次数 |
| 总失败数 | 响应状态码非 2xx 的请求数 |
| 成功率 | 成功请求占总请求的百分比 |
| 平均响应时间 | 所有请求响应时间的平均值（毫秒） |
| 最小响应时间 | 最快请求的响应时间（毫秒） |
| 最大响应时间 | 最慢请求的响应时间（毫秒） |
| P50 响应时间 | 50% 请求的响应时间阈值（毫秒） |
| P90 响应时间 | 90% 请求的响应时间阈值（毫秒） |
| P95 响应时间 | 95% 请求的响应时间阈值（毫秒） |
| P99 响应时间 | 99% 请求的响应时间阈值（毫秒） |
| RPS | 每秒处理请求数 |
| 错误分布 | 各类错误的发生次数统计 |

### 2.3 场景管理

#### 2.3.1 功能描述

管理压测场景，支持多种场景创建方式。

#### 2.3.2 功能列表

| 功能模块 | 功能描述 | 优先级 |
|----------|----------|--------|
| 场景列表 | 展示所有场景，支持筛选搜索 | 高 |
| 场景详情 | 展示场景的详细配置 | 高 |
| HAR 导入 | 上传 HAR 文件自动生成场景 | 高 |
| 自定义创建 | 手动创建场景和请求 | 高 |
| 场景编辑 | 编辑场景基本信息及请求配置 | 高 |
| 场景复制 | 复制已有场景创建新场景 | 中 |
| 场景删除 | 删除不需要的场景 | 中 |
| 批量操作 | 批量删除、复制、导出场景 | 低 |

#### 2.3.3 HAR 导入功能

支持以下 HAR 导入能力：

| 功能 | 说明 |
|------|------|
| 文件上传 | 支持上传 .har 格式文件 |
| 资源类型过滤 | 支持按 xhr、html、js、css、image、other 类型过滤 |
| Host 替换 | 支持替换请求的 Target Host |
| 场景命名 | 支持自定义场景名称 |
| 批量导入 | 支持一次导入多个 HAR 文件 |

#### 2.3.4 请求配置

每个请求应支持以下配置项：

| 配置项 | 类型 | 说明 |
|--------|------|------|
| 请求名称 | 字符串 | 接口名称标识 |
| HTTP 方法 | 枚举 | GET、POST、PUT、DELETE、PATCH |
| 请求 URL | 字符串 | 接口地址，支持变量替换 |
| 请求头 | JSON | HTTP 请求头配置 |
| 请求体 | 字符串 | POST/PUT 请求体内容 |
| 请求体类型 | 枚举 | 无、JSON、表单、文件 |
| 请求权重 | 整数 | 请求在场景中的权重比例 |
| 思考时间 | 浮点数 | 请求间隔时间（秒） |
| 超时时间 | 整数 | 请求超时时间（秒） |
| 数据源绑定 | JSON | 绑定数据源实现参数化 |

### 2.4 数据源管理

#### 2.4.1 功能描述

管理测试数据源，支持多种数据格式和生成方式。

#### 2.4.2 功能列表

| 功能模块 | 功能描述 | 优先级 |
|----------|----------|--------|
| 数据源列表 | 展示所有数据源 | 高 |
| 文件上传 | 上传 CSV、JSON 数据文件 | 高 |
| 数据库连接 | 连接 MySQL、PostgreSQL 等数据库 | 中 |
| 脚本生成 | 使用 Python 脚本生成测试数据 | 中 |
| 数据预览 | 预览数据源内容（前 N 行） | 高 |
| 测试连接 | 测试数据库连接是否正常 | 中 |
| 数据导出 | 导出数据源内容 | 低 |
| 数据删除 | 删除数据源 | 中 |

#### 2.4.3 支持的数据源类型

| 类型 | 说明 | 配置项 |
|------|------|--------|
| CSV | 逗号分隔值文件 | 文件路径、编码、分隔符 |
| JSON | JSON 格式数据 | 文件路径 |
| MySQL | MySQL 数据库 | host、port、user、password、database、query |
| PostgreSQL | PostgreSQL 数据库 | host、port、user、password、database、query |
| MongoDB | MongoDB 数据库 | host、port、user、password、database、collection |
| Redis | Redis 键值存储 | host、port、password、key、pattern |
| Python 脚本 | 自定义数据生成 | 脚本代码 |

#### 2.4.4 数据绑定

支持将数据源绑定到请求的以下部分：

| 绑定目标 | 说明 |
|----------|------|
| URL 参数 | 替换 URL 中的路径参数或查询参数 |
| 请求头 | 动态设置请求头，如 Token |
| 请求体 | 替换请求体中的占位符 |
| 文件上传 | 指定上传文件路径 |

### 2.5 压测图表

#### 2.5.1 功能描述

实时展示压测过程中的关键指标图表。

#### 2.5.2 图表类型

| 图表类型 | 用途 | 数据维度 |
|----------|------|----------|
| RPS 趋势图 | 展示每秒请求数变化 | 时间序列 |
| 响应时间趋势图 | 展示响应时间变化 | 时间序列、P50/P90/P99 |
| 累计请求图 | 展示累计请求数增长 | 时间序列 |
| 错误率趋势图 | 展示错误率变化 | 时间序列 |
| 并发用户数图 | 展示虚拟用户数变化 | 时间序列 |
| 错误分布图 | 展示各类错误的占比 | 饼图 |
| 请求分布热力图 | 展示请求响应时间分布 | 热力图 |

#### 2.5.3 实时监控

压测执行过程中应实时展示以下数据：

| 数据项 | 更新频率 | 说明 |
|--------|----------|------|
| 当前用户数 | 1 秒 | 当前活跃的虚拟用户数 |
| 总请求数 | 1 秒 | 已完成的请求总数 |
| RPS | 1 秒 | 当前每秒请求数 |
| 平均响应时间 | 1 秒 | 当前平均响应时间 |
| 错误率 | 1 秒 | 当前错误百分比 |
| 运行时间 | 实时 | 压测已运行时长 |

## 三、非功能需求

### 3.1 性能需求

| 需求项 | 要求 |
|--------|------|
| 页面响应时间 | 普通页面加载时间不超过 2 秒 |
| API 响应时间 | 简单查询 API 响应时间不超过 500 毫秒 |
| 并发用户数 | 支持至少 100 个并发用户访问管理界面 |
| 数据处理能力 | 单次压测支持百万级请求记录 |

### 3.2 安全需求

| 需求项 | 要求 |
|--------|------|
| 用户认证 | 支持用户注册、登录、登出 |
| 密码安全 | 密码必须加密存储，支持密码强度检测 |
| API 安全 | 支持 JWT Token 认证 |
| 数据隔离 | 用户只能访问自己的项目和资源 |
| 敏感数据 | Token、密码等敏感数据加密存储 |

### 3.3 可用性需求

| 需求项 | 要求 |
|--------|------|
| 系统可用性 | 年度可用性不低于 99% |
| 数据备份 | 定期备份数据库和上传文件 |
| 错误提示 | 友好的错误提示信息 |
| 操作确认 | 危险操作需要二次确认 |

### 3.4 兼容性需求

| 需求项 | 要求 |
|--------|------|
| 浏览器支持 | Chrome、Firefox、Edge、Safari 最新版本 |
| 响应式布局 | 支持 1920x1080、1366x768 等常见分辨率 |
| 移动端适配 | 提供基础的移动端适配 |
